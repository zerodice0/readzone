# Seed: Feed-First Main Page Redesign

## Goal

첫 접속 화면에서 설명형 랜딩 영역을 제거하고, 사용자가 바로 다른 독자의 글을 읽거나 자신의 독후감을 남길 수 있는 타임라인 중심 메인 페이지로 재구성한다.

## Problem

- 현재 메인 페이지의 큰 소개 패널이 모바일 첫 화면의 절반 이상을 차지한다.
- 서비스 정체성은 전달되지만, 실제 사용 목적인 읽기/쓰기/탐색으로 즉시 연결되지 않는다.
- 독자적인 브랜드 아이콘이 없어 첫 인상이 범용 템플릿처럼 보인다.
- 카드 그리드는 독후감 SNS보다 콘텐츠 갤러리에 가까워 글 흐름을 빠르게 훑기 어렵다.

## UX Direction

- Twitter/X의 정보 구조를 참고하되 시각 표현은 글다락의 종이/잉크 톤을 유지한다.
- 첫 화면 우선순위는 `작성 진입 -> 검색/필터 -> 최신 독후감`이다.
- 큰 히어로 대신 compact composer를 상단에 둔다.
- 피드는 단일 컬럼 타임라인으로 바꾸고, 각 항목은 작성자/책/본문/반응이 한 흐름으로 읽히게 한다.
- 독자적인 마크를 헤더와 composer에 배치해 브랜드 식별성을 보강한다.

## Discussion Decisions

- 비로그인 사용자의 작성 CTA는 유지하되, 클릭 시 기존 보호 라우트/로그인 흐름에 맡긴다.
- 현재 데이터 모델에 없는 추천 도서/트렌드 모듈은 만들지 않는다.
- 필터 기능은 기존 `recent/popular/recommended` 쿼리 구조를 그대로 사용한다.
- 리뷰 상세/작성 플로우는 이번 범위에서 변경하지 않는다.
- 모바일에서 가장 중요한 것은 "스크롤 없이 피드가 시작되는 느낌"이므로 장식성 문구는 최소화한다.

## Acceptance Criteria

- 모바일 첫 화면에서 거대한 소개 카드가 사라지고, 작성 진입과 피드 일부가 빠르게 보인다.
- 메인 페이지에 글다락 전용 브랜드 마크가 보인다.
- 피드 항목이 단일 타임라인 형태로 표시된다.
- 검색과 필터가 상단에서 접근 가능하며 기존 동작을 유지한다.
- TypeScript, ESLint, production build가 통과한다.
