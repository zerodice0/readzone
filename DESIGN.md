# DESIGN.md - 글다락 디자인 운영 기준

이 문서는 글다락의 화면 디자인, 색상 의미, 인터랙션 상태를 일관되게 유지하기 위한 기준이다.
프론트엔드 UI, 스타일, 레이아웃, 컴포넌트 상태를 변경할 때 먼저 이 문서를 확인한다.

## 핵심 원칙

- 화면은 읽기와 기록이라는 실제 사용 흐름을 먼저 보여준다. 큰 소개 영역보다 콘텐츠, 입력 진입점, 탐색 컨트롤을 우선한다.
- 종이 질감은 배경과 표면의 분위기다. 선택, 포커스, 오류, 성공 같은 의미 상태를 종이 색상으로 대신하지 않는다.
- 의미가 다른 색은 토큰도 분리한다. 같은 색이더라도 역할이 다르면 직접 재사용하지 않는다.
- 모바일에서는 탭 후 남는 포커스가 선택 상태처럼 보이지 않아야 한다. 키보드 접근성 포커스는 `focus-visible`을 기본으로 한다.

## 색상 역할

### Brand / Action

- 목적: 주요 CTA, 작성 버튼, 브랜드 포인트.
- 현재 구현: `primary`, `primary-*`.
- 사용 예: `오늘 기록`, `독후감 쓰기`, 저장, 작성 완료.
- 주의: `primary`를 모든 선택/포커스 상태에 자동으로 쓰지 않는다.

### Paper

- 목적: 종이 질감, 표면 배경, 얇은 경계, 책 표지 프레임.
- 현재 구현: `paper-*`, `paper-surface`, `paper-panel`, `paper-input`, `book-paper-frame`.
- 사용 예: 페이지 배경, 카드 표면, 구분선, 종이 질감.
- 금지: `paper-*`를 활성 탭, 현재 네비게이션, 포커스 링의 주 색으로 사용하지 않는다.

### Current / Selected

- 목적: 현재 라우트, 현재 탭, 선택된 필터, 선택된 날짜처럼 사용자가 선택한 상태.
- 원칙: 같은 종류의 선택 상태는 같은 색상 체계를 사용한다.
- 현재 기준: 라우트 네비게이션과 콘텐츠 내부의 날짜/필터/카드 선택 상태는 노란색 `primary` 계열로 통일한다.
- 세부 기준: 현재/선택 표시선은 `primary-500`, 선택 텍스트는 `primary-700`, 선택 배경은 `primary-50` 또는 `primary-100`을 우선한다.
- 날짜 셀처럼 터치 후 포커스가 남을 수 있는 UI는 `focus`가 아니라 명시적인 selected/current 상태에서만 선택 색을 보여주며, 어두운 갈색 `primary-800/900`이나 `paper` 계열을 주 선택색으로 쓰지 않는다.

### Focus

- 목적: 키보드 또는 보조기기 사용자가 현재 조작 위치를 알 수 있게 한다.
- 원칙: 포커스는 선택 상태가 아니다.
- 구현 기준:
  - 기본은 `focus-visible:*`를 사용한다.
  - `focus:*`는 입력 필드처럼 포커스 자체가 주요 상태인 컴포넌트에만 제한한다.
  - 전역 포커스 색은 `--ring` 의미 토큰으로만 관리한다.
  - 현재 `--ring`은 노란색 `primary-600` 계열이며, 화면별로 직접 색을 지정하지 않는다.
- 금지:
  - 날짜 셀, 카드, 네비게이션 링크에 `focus:ring-primary-*`를 직접 넣지 않는다.
  - 모바일 터치 후 남는 포커스가 선택된 항목처럼 보이게 만들지 않는다.

### Status

- 목적: 성공, 경고, 오류, 정보 상태.
- 기준:
  - 오류/삭제는 red 계열을 사용한다.
  - 정보성 강조는 `note-blue`, 보조 성공/안정 상태는 `ink-green`을 사용한다. 단, `note-blue`는 현재/선택/포커스 상태에 사용하지 않는다.
  - 상태 색을 브랜드 CTA 색으로 대체하지 않는다.

## 인터랙션 상태 규칙

하나의 컴포넌트에 아래 상태가 섞일 때 우선순위를 지킨다.

1. Disabled: 조작 불가 상태가 가장 우선한다.
2. Current / Selected: 현재 페이지, 선택된 탭, 선택된 필터.
3. Focus Visible: 키보드 접근성 표시.
4. Hover / Pressed: 임시 반응.
5. Default: 기본 상태.

상태별 사용 기준:

- `hover`: 마우스 반응. 모바일에서 핵심 의미를 hover에만 두지 않는다.
- `active`: 누르는 순간의 짧은 반응. 선택 상태처럼 오래 남기지 않는다.
- `focus-visible`: 키보드 포커스. 대부분의 버튼, 링크, 카드형 버튼의 기본 포커스 방식.
- `focus`: 텍스트 입력, textarea, 검색창 등 입력 맥락에서만 사용한다.
- `data-[state=active]`, `aria-current`, `aria-selected`: 현재/선택 상태를 표현할 때 사용한다.

## Tailwind 사용 규칙

- 공통 UI 컴포넌트는 가능하면 `components/ui`의 variant를 먼저 확장한다.
- 화면별 컴포넌트에서 색상을 직접 지정할 때는 역할을 먼저 판단한다.
- 새 디자인을 추가할 때 아래 직접 사용을 피한다.
  - `focus:ring-primary-*`
  - `focus-visible:ring-primary-*`를 여러 컴포넌트에 임의 반복
  - `ring-paper-*`를 포커스/선택 의미로 사용
  - `bg-paper-*`만으로 selected 상태 표현
- 반복되는 포커스/선택 조합은 유틸 클래스나 컴포넌트 variant로 승격한다.

권장 패턴:

```tsx
// 키보드 포커스만 표시
className="focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2"

// 현재 라우트/선택 상태는 명시 상태로 표현
className={isActive ? 'text-primary-700' : 'text-stone-500 hover:text-stone-800'}
```

피해야 할 패턴:

```tsx
// 모바일 탭 후 선택처럼 남을 수 있음
className="focus:ring-2 focus:ring-primary-500"

// 종이 배경 색을 선택 의미로 오용
className={isSelected ? 'bg-paper-100 ring-paper-200' : 'bg-white'}
```

## 종이 질감 테마 기준

- `paper-page`: 앱 전체 배경.
- `paper-surface`: 주요 표면, 리스트 컨테이너, 모달 표면.
- `paper-panel`: 반복 아이템, 보조 패널, 책/일기 조각.
- `paper-input`: 입력 필드.
- `book-paper-frame`: 책 표지 이미지 프레임.

종이 질감은 레이어가 많아질수록 화면이 답답해진다. 카드 안에 또 카드가 필요한 경우가 아니면 경계선과 간격으로 구분한다.

## 레이아웃 기준

- 메인 사용 흐름은 첫 화면에서 바로 보여준다.
- 피드형 화면은 `좌측 주요 콘텐츠 + 우측 보조 패널` 구조를 기본으로 한다.
- 모바일에서는 보조 정보보다 주요 액션과 콘텐츠를 먼저 배치한다.
- 고정 컨트롤은 현재 화면의 핵심 탐색에만 사용한다. 장식성 sticky 영역은 만들지 않는다.

## 디자인 변경 전 체크리스트

- 이 색은 브랜드, 종이, 선택, 포커스, 상태 중 어느 역할인가?
- 포커스와 선택 상태가 시각적으로 구분되는가?
- 모바일 터치 후 포커스가 선택처럼 남지 않는가?
- 같은 종류의 선택 상태가 네비게이션, 탭, 필터, 날짜 셀에서 같은 규칙을 쓰는가?
- 새 색상이나 새 radius를 추가하지 않고 기존 토큰으로 해결 가능한가?
- 반복되는 스타일이면 공통 컴포넌트/variant로 옮길 가치가 있는가?

## 현재 개선 대상

- `primary`가 브랜드 CTA와 선택/포커스 역할을 동시에 맡는 부분을 점진적으로 분리한다.
- `--ring`은 접근성 포커스 전용 의미로 유지하고, 화면별 직접 `focus:ring-primary-*` 사용을 줄인다.
- 날짜 셀, 탭, 네비게이션의 selected/current 표현은 `primary-50/500/700` 기준으로 통일한다.
